Deployment Guide

Deploy Buzz Notification Service in various environments.

Docker Compose (Development & Testing)

Quick Start

# Clone the repository
git clone https://github.com/elight/buzz-service.git
cd buzz-service

# Copy environment file
cp .env.example .env

# Edit configuration
nano .env

# Start all services
docker-compose up -d

# View logs
docker-compose logs -f buzz-service

docker-compose.yml Overview

The Docker Compose setup includes:

  • buzz-service: Main API server and workers
  • postgres: PostgreSQL 15 database
  • redis: Redis 7 for queuing

Useful Commands

# Stop all services
docker-compose down

# Restart specific service
docker-compose restart buzz-service

# View database logs
docker-compose logs postgres

# Execute database migrations
docker-compose exec buzz-service /app/buzz-service migrate

# Access PostgreSQL CLI
docker-compose exec postgres psql -U buzz -d buzzdb

Kubernetes (Production)

Prerequisites

  • Kubernetes cluster (1.24+)
  • kubectl configured
  • Helm 3 (optional, recommended)
  • External PostgreSQL database
  • External Redis instance or cluster

1. Create Namespace

kubectl create namespace buzz

2. Create Secrets

kubectl create secret generic buzz-secrets \
  --from-literal=db-password='YOUR_DB_PASSWORD' \
  --from-literal=redis-password='YOUR_REDIS_PASSWORD' \
  --from-literal=ses-access-key='YOUR_SES_KEY' \
  --from-literal=ses-secret-key='YOUR_SES_SECRET' \
  --from-literal=fcm-server-key='YOUR_FCM_KEY' \
  --namespace=buzz

3. Create ConfigMap

apiVersion: v1
kind: ConfigMap
metadata:
  name: buzz-config
  namespace: buzz
data:
  DB_HOST: "postgres.database.svc.cluster.local"
  DB_PORT: "5432"
  DB_NAME: "buzzdb"
  DB_USER: "buzz"
  REDIS_HOST: "redis.cache.svc.cluster.local"
  REDIS_PORT: "6379"
  SERVER_PORT: "8080"
  LOG_LEVEL: "info"
  ENVIRONMENT: "production"

4. Deploy Application

apiVersion: apps/v1
kind: Deployment
metadata:
  name: buzz-api
  namespace: buzz
spec:
  replicas: 3
  selector:
    matchLabels:
      app: buzz-api
  template:
    metadata:
      labels:
        app: buzz-api
    spec:
      containers:
      - name: buzz
        image: ghcr.io/elight/buzz-service:latest
        ports:
        - containerPort: 8080
        envFrom:
        - configMapRef:
            name: buzz-config
        env:
        - name: DB_PASSWORD
          valueFrom:
            secretKeyRef:
              name: buzz-secrets
              key: db-password
        - name: REDIS_PASSWORD
          valueFrom:
            secretKeyRef:
              name: buzz-secrets
              key: redis-password
        resources:
          requests:
            memory: "256Mi"
            cpu: "250m"
          limits:
            memory: "512Mi"
            cpu: "500m"
        livenessProbe:
          httpGet:
            path: /health
            port: 8080
          initialDelaySeconds: 10
          periodSeconds: 10
        readinessProbe:
          httpGet:
            path: /health
            port: 8080
          initialDelaySeconds: 5
          periodSeconds: 5

5. Deploy Workers

apiVersion: apps/v1
kind: Deployment
metadata:
  name: buzz-workers
  namespace: buzz
spec:
  replicas: 5
  selector:
    matchLabels:
      app: buzz-workers
  template:
    metadata:
      labels:
        app: buzz-workers
    spec:
      containers:
      - name: worker
        image: ghcr.io/elight/buzz-service:latest
        command: ["/app/buzz-service", "worker"]
        envFrom:
        - configMapRef:
            name: buzz-config
        env:
        - name: DB_PASSWORD
          valueFrom:
            secretKeyRef:
              name: buzz-secrets
              key: db-password
        resources:
          requests:
            memory: "256Mi"
            cpu: "200m"
          limits:
            memory: "512Mi"
            cpu: "400m"

6. Create Service & Ingress

apiVersion: v1
kind: Service
metadata:
  name: buzz-api
  namespace: buzz
spec:
  selector:
    app: buzz-api
  ports:
  - port: 80
    targetPort: 8080
  type: ClusterIP
---
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: buzz-ingress
  namespace: buzz
  annotations:
    cert-manager.io/cluster-issuer: "letsencrypt-prod"
spec:
  ingressClassName: nginx
  tls:
  - hosts:
    - notifications.yourdomain.com
    secretName: buzz-tls
  rules:
  - host: notifications.yourdomain.com
    http:
      paths:
      - path: /
        pathType: Prefix
        backend:
          service:
            name: buzz-api
            port:
              number: 80

Configuration

Environment Variables

Configure the service using environment variables:

# Database
DB_HOST=localhost
DB_PORT=5432
DB_NAME=buzzdb
DB_USER=buzz
DB_PASSWORD=your_secure_password
DB_SSL_MODE=disable

# Redis
REDIS_HOST=localhost
REDIS_PORT=6379
REDIS_PASSWORD=
REDIS_DB=0

# Server
SERVER_PORT=8080
SERVER_READ_TIMEOUT=30s
SERVER_WRITE_TIMEOUT=30s

# Logging
LOG_LEVEL=info
LOG_FORMAT=json

# Email Providers
EMAIL_PROVIDER=ses
SES_REGION=us-east-1
SES_ACCESS_KEY_ID=your_access_key
SES_SECRET_ACCESS_KEY=your_secret_key
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USERNAME=your_email@gmail.com
SMTP_PASSWORD=your_app_password

# SMS Providers
SMS_PROVIDER=notifylk
NOTIFYLK_API_KEY=your_notifylk_key
NOTIFYLK_USER_ID=your_user_id
TWILIO_ACCOUNT_SID=your_twilio_sid
TWILIO_AUTH_TOKEN=your_twilio_token
TWILIO_FROM_NUMBER=+1234567890

# Push Notifications
FCM_SERVER_KEY=your_fcm_server_key
FCM_PROJECT_ID=your_firebase_project

# Worker Configuration
WORKER_CONCURRENCY=10
WORKER_QUEUES=default,high-priority,low-priority

# Rate Limiting
RATE_LIMIT_ENABLED=true
DEFAULT_RATE_LIMIT_PER_MINUTE=100

Provider Setup

Amazon SES Setup

  1. Create AWS account and navigate to SES
  2. Verify your sending domain
  3. Create IAM user with SES send permissions
  4. Generate access keys
  5. Add keys to configuration
EMAIL_PROVIDER=ses
SES_REGION=us-east-1
SES_ACCESS_KEY_ID=AKIAIOSFODNN7EXAMPLE
SES_SECRET_ACCESS_KEY=wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY

NotifyLK Setup (Sri Lanka SMS)

  1. Register at notify.lk
  2. Get API credentials
  3. Add to configuration
SMS_PROVIDER=notifylk
NOTIFYLK_API_KEY=your_api_key
NOTIFYLK_USER_ID=your_user_id
NOTIFYLK_SENDER_ID=YourBrand

Firebase Cloud Messaging Setup

  1. Create Firebase project
  2. Enable Cloud Messaging
  3. Get server key from Project Settings
  4. Add to configuration
FCM_SERVER_KEY=AAAA1234567890abcdefghijklmnopqrstuvwxyz
FCM_PROJECT_ID=your-project-id

Scaling Guidelines

Horizontal Scaling

Scale components independently based on load:

# Scale API servers
kubectl scale deployment buzz-api --replicas=10 -n buzz

# Scale workers
kubectl scale deployment buzz-workers --replicas=20 -n buzz

Database Scaling

  • Use read replicas for query scaling
  • Enable connection pooling (PgBouncer)
  • Implement table partitioning for large tables
  • Regular vacuuming and index maintenance

Redis Scaling

  • Use Redis Cluster for high availability
  • Separate caching and queuing instances
  • Monitor memory usage and eviction policies

Performance Benchmarks

Configuration Throughput Latency (p95)
Single API + 5 Workers ~1,000/min 50ms
3 API + 20 Workers ~10,000/min 45ms
10 API + 50 Workers ~50,000/min 40ms
Production Checklist
  • Enable HTTPS with valid SSL certificates
  • Set up database backups and disaster recovery
  • Configure Redis persistence (AOF or RDB)
  • Implement log aggregation (ELK, Loki, etc.)
  • Set up monitoring and alerting
  • Review and set appropriate rate limits
  • Secure secrets management (Vault, AWS Secrets Manager)
  • Network security and firewall rules
  • Regular security updates
