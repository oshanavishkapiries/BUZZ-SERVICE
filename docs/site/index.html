Buzz Notification Service

Unified multi-channel notification delivery platform

Send emails, SMS, push notifications, and in-app messages with a single API

Get Started API Reference

Features

Multi-Channel

Send notifications via Email (SES/SMTP), SMS (NotifyLK/Twilio), Push (FCM), and In-App messaging.

High Performance

Queue-based architecture with Redis and worker pools for scalable notification delivery.

Templates

Reusable notification templates with variable substitution and multi-channel support.

Bulk Sending

Send to thousands of recipients from datasources like Google Sheets, CSV, or APIs.

Real-Time

Server-Sent Events (SSE) for instant in-app notification delivery.

Observable

Prometheus metrics, structured logging, and comprehensive tracking.

Quick Start

1

Install Dependencies

# Using Docker Compose (recommended)
docker-compose up -d

# Or manually
go mod download
2

Configure Environment

# Copy example configuration
cp .env.example .env

# Edit configuration
nano .env
3

Start the Service

# Using Docker
docker-compose up -d

# Or using Make
make run
4

Send Your First Notification

curl -X POST http://localhost:8080/api/v1/notifications \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "channel": "email",
    "recipient": {
      "email": "user@example.com"
    },
    "subject": "Welcome!",
    "body": "Hello from Buzz!"
  }'
